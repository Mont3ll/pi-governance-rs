# Agent Instructions for PI

When starting a non-trivial coding task:

1. Retrieve PI context for the project.
2. Prefer active records.
3. Treat contested records as unsafe unless explicitly included.
4. If the user corrects you, propose a correction memory.
5. If a memory is stale, contest or supersede it.
6. Do not silently store high-impact memories under strict policy.
7. Review pending patches before release work.

PI complements codebase intelligence MCP servers and skill libraries. It does not replace them.
