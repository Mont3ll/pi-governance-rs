PI Governance Rust Port

Local-first governed memory for coding agents

Current stable release: v1.0.0. Stable v1.0.0 has shipped.

What PI is

PI Governance is a local-first runtime that gives coding agents durable memory without silent writes. Memory changes are proposed as patches, reviewed, applied under policy, and kept auditable in a JSONL source-of-truth store.

First 10 minutes

cargo build -p pi-governance-rs
./target/debug/pi demo
./target/debug/pi --store /tmp/pi-governance-demo review
./target/debug/pi --store /tmp/pi-governance-demo retrieve "release workflow" --explain
./target/debug/pi --store /tmp/pi-governance-demo list
./target/debug/pi --store /tmp/pi-governance-demo inspect-record <record_id>
./target/debug/pi mcp-config opencode --command /path/to/pi --store /path/to/.pi --namespace default
./target/debug/pi mcp-install opencode --command /path/to/pi --store /path/to/.pi --namespace default
./target/debug/pi mcp-doctor opencode --command /path/to/pi --store /path/to/.pi --namespace default
./target/debug/pi agent-instructions

Governance lifecycle

Patch before mutation

Durable records enter the store through proposed patches, not silent background writes.

Review

pi review lists and inspects pending patches before they are applied.

Policy

Profiles can allow, warn, or require manual review while preserving existing governance semantics.

Audit

Applied, contested, superseded, and tombstoned records preserve history instead of erasing context.

Core capabilities

  • Review/demo/inspect-record/agent-instructions: quick onboarding, auditability, and coding-agent guidance.
  • Policy profiles: permissive, standard, and strict operation modes.
  • Namespace isolation: logical separation for projects or agents while keeping JSONL storage.
  • Belief revision: contest, supersede, tombstone, reinforce, and resolve contested records.
  • Import/export: portable JSON bundles with preview and backup options.
  • MCP stdio integration: coding agents can retrieve governed memory and propose changes.
  • MCP setup helpers: pi mcp-config opencode/codex/pi-agent, pi mcp-install, and pi mcp-doctor streamline client registration. Some clients display prefixed tool names such as pi-governance_pi_retrieve_context or pi_governance_pi.retrieve_context; these are equivalent to pi.retrieve_context. The pi.inspect_record MCP tool remains deferred unless implemented separately.

Security and memory-poisoning limits

PI reduces memory-poisoning risk by making durable memory writes explicit, inspectable, policy-controlled, revisable, and auditable. It does not prove that accepted memories are true, prevent every prompt injection, encrypt local JSONL files by default, or sandbox arbitrary tools.

Ecosystem fit

For public testing, start with docs/PUBLIC_TESTING.md, docs/NON_GOALS.md, and docs/MCP_TROUBLESHOOTING.md. PI complements codebase intelligence MCP servers and skill libraries. Codebase-memory tools answer structural questions about code. Skill libraries package reusable workflows. PI stores governed decisions, corrections, workflows, and revisions across sessions.

pi-governance-rs is the standalone Rust CLI/MCP runtime. pi-persistent-intelligence is the standalone lightweight pi-agent-native extension. Either can be used alone; together they interoperate through the shared PI memory contract and import/export bundles.

Portable Workflow Parity

v1.0.0 includes deterministic memory-worth scoring, capture candidates, inbox review, scoped context output, session search/decisions, recall X-ray, explicit L1/L2/L3 layers, and trust/durability/source verification gates. Capture creates candidates or L3 evidence only; durable L1/L2 records remain patch-governed.
